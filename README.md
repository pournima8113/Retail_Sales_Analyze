# Retail_Sales_Analyze

## 📌 Project Overview
This project is a comprehensive SQL-based solution designed to manage and analyze retail operations. It covers the entire lifecycle of a retail transaction—from Inventory Management and Customer Loyalty to Salesperson Performance and Payment Mode Trends. The goal is to provide actionable business intelligence through structured data analysis.

## 🚀 Key Business Features
- Sales Tracking: Records real-time sales with connections to customers and staff.
- Staff Analytics: Tracks sales performance of employees (Male/Female) to identify top performers.
- Payment Gateway Analysis: Monitors revenue distribution across Online, Cash, Card, and UPI.
- Returns & Exchanges: A dedicated system to track product returns and the reason for exchange/refund.
- Marketing & Offers: Integrated promotional tracking (e.g., Diwali Sale) to see how discounts affect final pricing.
- Customer Loyalty: A point-based system to reward frequent shoppers.

## 🛠️ Technical Architecture (Database Schema)
The database is structured into 7 normalized tables to ensure data integrity and avoid redundancy:
- Customers: Stores unique customer details and loyalty points.
- Employees: Tracks staff information, including gender for performance demographic analysis.
- Products: Inventory management with Cost Price, Selling Price, and Stock levels.
- Orders: The central fact table connecting customers, staff, and products.
- Payments: Detailed tracking of transaction methods.
- Offers: Stores promotional data linked to specific products.
- Returns: Handles post-sales support and inventory adjustments.

## 📊 Analytics & Insights (Queries Included)
The project includes advanced SQL queries to generate the following reports:
- Revenue by Payment Method: Highlighting which platform (like UPI or Card) is most used.
- Employee Leaderboard: Total sales generated by each salesperson.
- Gender-based Sales Trends: Understanding if sales vary based on staff demographics.
- Profit Margin Analysis: Calculating net profit per product after discounts.

## 📁 File Structure
To keep the project organized and professional, the code is divided into:
- schema.sql: Contains the DDL (Data Definition Language) for creating tables.
- data.sql: Contains the DML (Data Manipulation Language) with sample records.
- queries.sql: Contains the analytical queries used for business reporting.

## 💻 How to Run This Project
- Clone this repository or download the files.
- Open your MySQL Client (MySQL Workbench, VS Code, etc.).
- Execute schema.sql to build the database structure.
- Execute data.sql to populate the tables with sample data.
- Run queries.sql to see the analytical reports and results.

## 🧠 Database Concepts Implemented
- **3NF Normalization:** Ensured zero data redundancy.
- **Relational Mapping:** Established One-to-Many and Many-to-Many relationships.
- **Data Integrity:** Used constraints like `NOT NULL`, `UNIQUE`, and `FOREIGN KEY`.
- **Complex Aggregations:** Performed multi-table joins to generate business KPIs.
